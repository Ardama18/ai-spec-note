# サムネイル・ビジュアル戦略 - note×AI最適化統合デザイン

このドキュメントは、noteプラットフォームとAI最適化を統合したサムネイル戦略を定義します。

## 📐 技術仕様とプラットフォーム要件

### note OGP画像の基本仕様

**推奨サイズ**:
- **幅×高さ**: 1280×670ピクセル
- **アスペクト比**: 1.91:1（約2:1）
- **最小サイズ**: 600×315ピクセル
- **最大ファイルサイズ**: 1MB以下推奨

**ファイル形式**:
- **推奨**: PNG（透過なし）
- **代替**: JPG（品質90%以上）
- **非推奨**: GIF、WebP（一部環境で表示されない）

**カラースペース**:
- **標準**: sRGB
- **カラーモード**: RGB
- **ビット深度**: 8bit/チャンネル

### モバイル表示最適化

**スマートフォン表示考慮**:
- noteユーザーの80%がモバイル閲覧
- 最小表示サイズ: 375×197ピクセル（iPhone SE）
- タップ可能領域: 画面幅の100%

**テキスト可読性基準**:
- **最小フォントサイズ**: 48pt以上（モバイル判読可能）
- **推奨フォントサイズ**: 64-80pt（メインコピー）
- **行間**: 1.2-1.4倍（可読性確保）
- **文字数制限**: メインコピー12-15文字以内

**視認性確保**:
- コントラスト比: 4.5:1以上（WCAG AA基準）
- 背景とテキストの明度差: 十分な差を確保
- シャドウ効果: 必要に応じて追加

## 🎨 デザイン戦略フレームワーク

### 3秒ルール - 瞬時価値伝達

**原則**: タイムラインで3秒以内に記事価値を伝達

**実装方法**:
1. **メインメッセージの明確化**: 核心価値を1フレーズで表現
2. **視覚的階層**: 重要度順に情報配置
3. **色彩心理**: ターゲット感情に合わせた配色

**良い例**:
```
メインコピー（大）: 「月10万円達成」
サブコピー（小）: 「副業3ヶ月の全記録」
ビジュアル: グラフ・数字の視覚化
```

**悪い例**:
```
テキストのみ: 「副業を始めたい人のための詳細ガイド完全版」
（長すぎる、具体性なし、ビジュアル不足）
```

### ビジュアルスタイル分類

#### 1. 情報提供型（Data-Driven）
**適用記事**: ハウツー、ガイド、リスト記事

**デザイン特徴**:
- アイコン・図表の活用
- 数字・統計の視覚化
- 清潔感のあるレイアウト
- 寒色系（青・緑）の配色

**配色例**:
- メイン: #2196F3（青）
- アクセント: #FFC107（黄）
- 背景: #FFFFFF（白）

**テンプレート要素**:
- チェックリストアイコン
- ステップ番号
- プロセス矢印

#### 2. 感情訴求型（Emotional）
**適用記事**: 体験談、ストーリー、感動系

**デザイン特徴**:
- 人物写真・イラスト
- 暖色系（赤・オレンジ）の配色
- 柔らかいグラデーション
- 感情を呼ぶビジュアル

**配色例**:
- メイン: #FF6B6B（コーラルレッド）
- アクセント: #4ECDC4（ターコイズ）
- 背景: #FFE5E5（ライトピンク）

**テンプレート要素**:
- 吹き出し
- ハートマーク
- 感情アイコン

#### 3. 実績提示型（Achievement）
**適用記事**: 成果報告、ケーススタディ、実績紹介

**デザイン特徴**:
- 数字の大胆な使用
- グラフ・チャート
- ビフォーアフター比較
- 中立色+アクセントカラー

**配色例**:
- メイン: #424242（ダークグレー）
- アクセント: #00BCD4（シアン）
- 数字強調: #FF5722（オレンジレッド）

**テンプレート要素**:
- 大きな数字表示
- 上昇矢印
- トロフィーアイコン

#### 4. ストーリー型（Narrative）
**適用記事**: 連載、シリーズ、ドキュメンタリー

**デザイン特徴**:
- シーンイメージ
- 映画風構図
- ドラマチックな照明効果
- 暗めのトーン

**配色例**:
- メイン: #37474F（ブルーグレー）
- アクセント: #FFD54F（ゴールド）
- 背景: #263238（ダークブルーグレー）

**テンプレート要素**:
- シネマティックフレーム
- グラデーションオーバーレイ
- ドラマチックタイポグラフィ

## 🧠 AI最適化連携戦略

### AEO（Answer Engine Optimization）統合

**サムネイルでのAEO実装**:
1. **質問形式の視覚化**: 「どうやって？」「なぜ？」を明示
2. **回答の示唆**: サムネイルで答えの方向性を示す
3. **引用されやすいフレーズ**: 短く明確な言葉選び

**実装例**:
```
質問: 「副業で挫折しない秘訣は？」
回答示唆: 「3つの継続メソッド」
視覚要素: 3つのステップアイコン
```

### GEO（Generative Engine Optimization）統合

**生成AI検索対応**:
1. **要約可能なメッセージ**: AIが要約しやすい構造
2. **キーワードの視覚化**: 重要キーワードをテキストで明示
3. **階層的情報配置**: メイン→サブ→詳細の順

**実装例**:
```
メインメッセージ: 「月10万円達成」（大）
キーワード: 「副業」「プログラミング」（中）
詳細情報: 「3ヶ月の記録」（小）
```

### SEO統合効果

**検索エンジンでの効果**:
- **CTR向上**: 視覚的訴求による15-30%のクリック率改善
- **滞在時間増加**: 期待と実際の内容の一致で離脱率低下
- **SNS拡散促進**: 共有時のOGP表示最適化

## 🎯 noteプラットフォーム特化戦略

### タイムライン最適化

**note特有の表示環境**:
- タイムライン表示: 横長の矩形（1.91:1）
- 記事一覧: グリッド表示（正方形トリミング考慮）
- 検索結果: 小さいサムネイル表示

**セーフゾーン設計**:
```
┌─────────────────────────────┐
│  【Danger Zone】            │  上下50px
│                             │
│  ┌───────────────────┐      │
│  │                   │      │
│  │   Safe Zone       │      │  重要要素を配置
│  │   (Center 70%)    │      │
│  │                   │      │
│  └───────────────────┘      │
│                             │
│  【Danger Zone】            │  トリミングリスク
└─────────────────────────────┘
```

**配置戦略**:
- **中央70%**: メインコピー・重要ビジュアル
- **上下余白**: 装飾要素のみ（トリミングされても問題なし）

### ブランド一貫性

**シリーズ記事のビジュアル統一**:
1. **カラースキーム統一**: シリーズ全体で共通の配色
2. **フォント統一**: タイポグラフィの一貫性
3. **レイアウトパターン**: 配置ルールの共通化

**ブランド要素配置**:
- ロゴ位置: 右下または左上（10%領域内）
- アイコン: 統一されたビジュアルスタイル
- ブランドカラー: 必ず1色以上使用

## 🛠️ Canva活用ベストプラクティス

### テンプレート選択基準

**noteプラットフォーム向けテンプレート**:
1. **Canvaテンプレート検索**: "Blog Banner 1280x670"
2. **カスタムサイズ作成**: 1280x670pxで新規デザイン
3. **既存テンプレート改変**: アスペクト比調整

**テンプレートフィルタリング**:
- シンプルなレイアウト優先
- テキスト編集が容易
- 要素の移動・削除が簡単

### デザイン要素のカスタマイズ

**テキスト要素**:
```yaml
メインコピー:
  フォントサイズ: 64-80pt
  フォント: 太字・ゴシック系
  配置: 中央または上部1/3
  色: 高コントラスト

サブコピー:
  フォントサイズ: 36-48pt
  フォント: 中字・同系統
  配置: メインコピーの下または横
  色: アクセントカラー
```

**ビジュアル要素**:
- **写真・イラスト**: Canva内蔵素材またはアップロード
- **アイコン**: シンプルで認識しやすいもの
- **図形**: 情報を視覚化する補助要素

**カラーパレット作成**:
1. メインカラー: 記事テーマに合わせた主色
2. アクセントカラー: 注目を集める対比色
3. 背景カラー: 可読性を高める中立色

### エクスポート設定

**最適な出力設定**:
```yaml
ファイル形式: PNG
品質: 高品質（100%）
サイズ: 1280x670px（変更なし）
圧縮: 軽度（1MB以下に収める）
```

**ファイル命名規則**:
- パターン: `[記事ID]-thumbnail.png`
- 例: `C001-01-thumbnail.png`

## 📊 効果測定とA/Bテスト

### KPI指標

**サムネイル効果指標**:
1. **CTR（クリック率）**: タイムライン表示からのクリック率
2. **滞在時間**: 記事への誘導後の平均滞在時間
3. **エンゲージメント率**: いいね・コメント・共有の総合指標
4. **SNS拡散数**: Twitter/Facebookでの共有数

**測定方法**:
- noteアナリティクス（公式機能）
- Google Analytics（外部連携）
- SNSシェアカウント

### A/Bテスト戦略

**テストパターン例**:

**テスト1: タイトル配置**
- パターンA: 中央配置
- パターンB: 上部1/3配置
- 測定: CTR比較

**テスト2: カラースキーム**
- パターンA: 寒色系（青・緑）
- パターンB: 暖色系（赤・オレンジ）
- 測定: エンゲージメント率比較

**テスト3: ビジュアルスタイル**
- パターンA: 写真メイン
- パターンB: アイコン・図表メイン
- 測定: 滞在時間比較

**テスト実施プロセス**:
1. 仮説設定（どちらが効果的か）
2. パターン作成（2つのサムネイル）
3. 期間設定（最低1週間）
4. データ収集（note analytics）
5. 結果分析（統計的有意差確認）
6. 勝ちパターン採用
