# コンテンツ戦略作成基準

## 企画判定マトリクス

| 条件 | 必要ドキュメント | 作成順序 |
|-----|--------------|---------|
| 新規テーマ・分野 | コンテンツ企画書 → 構成案 → 投稿計画書 | 企画書承認後 |
| 戦略変更該当（下記参照） | 戦略書 → 構成案 → 投稿計画書 | 即座に開始 |
| シリーズ記事（6記事以上） | 戦略書 → 構成案 → 投稿計画書（必須） | 即座に開始 |
| 関連記事群（3-5記事） | 構成案 → 投稿計画書（推奨） | 即座に開始 |
| 単発記事（1-2記事） | なし | 直接執筆 |

## 戦略変更条件（いずれか該当で必須）

### 1. ターゲット読者変更
- **3つ以上の読者層の追加**: 既存読者層に加えて新しいペルソナ設定
  - 判断理由: 読者層の多様化はコンテンツ戦略の根本的変更
- **メイン読者層の変更・削除**
  - 判断理由: 複数記事への影響は慎重な判断が必要
- **読者の専門性レベル変更**（例: 初心者向け→専門家向け）
  - 判断理由: 文章レベルと内容深度の変更は戦略思想に関わる

### 2. コンテンツフロー変更
- **配信チャネル変更**（note単体→SNS連携、有料→無料）
- **3ステップ以上の投稿順序変更**
  - 例: 「基礎→応用→実践」から「実践→基礎→応用」
- **エンゲージメント手法変更**（コメント重視→シェア重視）

### 3. 収益化戦略変更
- モデル追加・変更・コンテンツ課金体系再配置

### 4. 外部連携変更
- SNSプラットフォーム・外部サービス・APIツール導入・置換

### 5. 複雑なコンテンツ設計（規模に関わらず）
- 3つ以上のトピックを統合管理
- 5つ以上のプラットフォームでの同時最適化

## 各ドキュメントの詳細定義

### コンテンツ企画書

**目的**: コンテンツのビジネス価値と読者価値を定義

**含むもの**:
- ターゲット読者とそのニーズ
- 成功指標とKPI（測定可能な形式）
- 読者ジャーニーとユースケース
- MoSCoW法による優先順位（Must/Should/Could/Won't）
- MVPコンテンツとFutureコンテンツの分離
- 読者ジャーニー図（必須）
- コンテンツスコープ境界図（必須）

**含まないもの**:
- 具体的な記事構成（→構成案）
- 技術的最適化手法（→投稿計画書）
- **投稿スケジュール**（→投稿計画書）
- **詳細タスク分解**（→投稿計画書）

### 戦略書（Content Strategy Record）

**目的**: コンテンツ戦略決定の理由と背景を記録

**含むもの**:
- 決定事項（どの戦略を選択したか）
- 根拠（なぜその戦略を選択したか）
- 選択肢の比較（最低3案）とトレードオフ
- ブランド戦略への影響
- コンテンツ制作への原則的な指針（例:「ファクトベース重視」）

**含まないもの**:
- 制作スケジュール、期間（→投稿計画書）
- 制作手順の詳細（→構成案）
- 具体的な文章例（→構成案）
- 担当者の割り当て（→投稿計画書）

### 構成案

**目的**: コンテンツの具体的構造を詳細定義

**含むもの**:
- **既存コンテンツ分析**（必須）
  - コンテンツマッピング（既存と新規の両方を記載）
  - 統合点の明確化（新規コンテンツでも既存との関連点を記載）
- AEO/GEO/SEO最適化アプローチ
- **構造的依存関係と制作制約**（制作の必要順序）
- 見出し構成と情報設計
- データフローとコンテンツ設計
- **読者エンゲージメント確認手順**
- **受入条件（測定可能な形式）**
- 変更影響マップ（直接影響/間接影響/波及なしを明記）
- 関連コンテンツとの統合点
- 情報の一貫性確保
- **合意事項チェックリスト**（ステークホルダーとの合意内容）
- **前提となる戦略書**（共通戦略含む）

**含まないもの**:
- なぜその戦略を選んだか（→戦略書参照）
- いつ制作するか、期間（→投稿計画書）
- 誰が制作するか（→投稿計画書）

### 投稿計画書

**目的**: コンテンツ制作タスクの管理と進捗追跡

**含むもの**:
- **フェーズ構成**（構成案の構造的依存関係を基に作成）
- タスク分解と依存関係（最大2階層まで）
- スケジュールと期間見積もり
- **構成案のエンゲージメント確認手順を各フェーズに配置**
- **最終フェーズに品質保証を含む**（必須）
- 進捗記録（チェックボックス形式）

**含まないもの**:
- 戦略的な根拠（→戦略書）
- 構成の詳細（→構成案）
- 構造的依存関係の決定（→構成案）

**タスク完了定義の3要素**:
1. **制作完了**: コンテンツが完成している
2. **品質完了**: AEO/GEO/SEO・読みやすさ・エンゲージメントがクリア
3. **統合完了**: 他コンテンツとの連携確認

## 作成プロセス

1. **課題分析**: コンテンツ規模判定、戦略変更条件確認
2. **戦略選択肢検討**（戦略書時のみ）: 3案以上比較、トレードオフ明記
3. **作成**: テンプレート使用、測定可能な条件記載
4. **承認**: レビュー後「Accepted」で制作開始

## 保存場所

| ドキュメント | パス | 命名規則 |
|------------|-----|---------|-
| コンテンツ企画書 | `content/planning/` | `[ID]-[テーマ名]-planning.md` | 
| 戦略書 | `content/strategy/` | `[ID]-[テーマ名]-strategy-[連番].md` |
| 構成案 | `content/structure/` | `[ID]-[テーマ名]-structure.md` |
| 投稿計画書 | `content/schedule/` | `[ID]-[テーマ名]-schedule.md` | 

※投稿計画書は`.gitignore`で除外

## 戦略書ステータス
`Proposed` → `Accepted` → `Deprecated`/`Superseded`/`Rejected`

## AI自動化ルール
- 6記事以上: 戦略書作成提案
- ターゲット読者・収益化モデル変更検出: 戦略書必須化
- 既存戦略書確認してから制作開始

## 図表作成要件

各ドキュメントで必須の図表（mermaid記法使用）：

| ドキュメント | 必須図表 | 目的 |
|------------|---------|-----|
| コンテンツ企画書 | 読者ジャーニー図、コンテンツスコープ境界図 | 読者体験と範囲の明確化 |
| 戦略書 | 戦略選択肢比較図（必要時） | トレードオフの視覚化 |
| 構成案 | コンテンツ構造図、情報フロー図 | 構造の理解とAI最適化 |
| 投稿計画書 | フェーズ構成図、タスク依存関係図 | 制作順序の明確化 |

## 共通戦略との関係性
1. **作成時**: 共通コンテンツ領域（SEO、AEO、GEO等）を特定し、既存共通戦略を参照
2. **不足時**: 必要な共通戦略が存在しない場合は作成を検討
3. **構成案**: 「前提となる戦略書」セクションで共通戦略を明記
4. **準拠確認**: 構成が共通戦略の決定事項と整合しているかを検証
